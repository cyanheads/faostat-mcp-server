# 0.1.5 — 2026-06-20

Framework maintenance only — no tool or behavior changes.

## Added

- **`scripts/check-dependency-specifiers.ts` + `Dependency Specifiers` devcheck step** — rejects floating specifiers (`latest`, `*`, pre-release dist-tags) in `package.json`'s dependency sections and `bun.lock`'s `workspaces` specifier map, so a `bun update --latest` that writes a literal `latest` into the lock can no longer ship unnoticed and re-resolve a dep past its pinned range. `peer`/`optional` may still float to `*`/dist-tags; `latest` never passes. ([mcp-ts-core#246](https://github.com/cyanheads/mcp-ts-core/issues/246))

## Changed

- **`Packaging` devcheck step now also runs on plugin manifests** — `scripts/lint-packaging.ts` validates `.claude-plugin/plugin.json` and `.codex-plugin/plugin.json`/`mcp.json` (non-empty descriptions; display fields carry the unscoped machine name while the `npx -y` install arg carries the full scoped package name), gated behind `packaging.pluginManifests` in `devcheck.config.json`. The step previously ran only when `manifest.json` was present. ([mcp-ts-core#240](https://github.com/cyanheads/mcp-ts-core/issues/240))
- **`skills/polish-docs-meta` re-synced** to 2.9 — documents the plugin-manifest lint guard and the scoped-vs-unscoped identity split it enforces.
