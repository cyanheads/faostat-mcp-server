# 0.1.6 — 2026-07-13

## Added

- **`truncated` — `faostat_dataframe_query` output field.** Boolean; `true` when `row_limit` capped the result and more rows exist than were returned. Surfaced in `structuredContent`, `format()`, and the enrich notice, with recovery guidance (page with `ORDER BY` + SQL `LIMIT`/`OFFSET`, raise `row_limit`, or aggregate). ([#10](https://github.com/cyanheads/faostat-mcp-server/issues/10))
- **`canvas_not_found` — typed `NotFound` on `faostat_dataframe_query` and `faostat_dataframe_describe`.** An explicit `canvas_id` that does not resolve to a live canvas (unknown, expired, or another tenant's) now fails typed instead of silently falling back to the session canvas. ([#11](https://github.com/cyanheads/faostat-mcp-server/issues/11))
- **Supply-chain install guards** — `bunfig.toml` adds a 3-day `minimumReleaseAge` (the framework dep excepted) and a Socket `[install.security]` scanner (`@socketsecurity/bun-security-scanner`).
- **Repo metadata** — `.github/SECURITY.md`, `.github/FUNDING.yml`, `.gitattributes`, and a `license` field on `manifest.json`.

## Changed

- **`row_count` — `faostat_dataframe_query`.** Documented as the returned/materialized count (equal to `rows.length`); when `truncated` is `true` this is explicitly NOT the full result total (no exact total is computed on this path). ([#10](https://github.com/cyanheads/faostat-mcp-server/issues/10))
- **`canvas_id` is honored on the dataframe tools.** `faostat_dataframe_describe` scopes its table listing to the resolved canvas; `StagedTableMeta` now carries `canvasId`. ([#11](https://github.com/cyanheads/faostat-mcp-server/issues/11))
- **Docker build** — base image pinned to `oven/bun:1.3.14`; `bun install --ignore-scripts` skips unused dependency postinstalls (`@duckdb/node-api` ships prebuilt binaries; `better-sqlite3`'s node-gyp build is unused under Bun). Framework `scripts/` and `skills/` re-synced to mcp-ts-core 0.10.14.

## Fixed

- **`faostat_dataframe_query` capped results advertised as complete.** The truncation check read a dead `rowCount > rows.length` comparison — always false on this path, where both are capped at `row_limit` — so a capped result disclosed no cap. It now reads the framework's `QueryResult.truncated` flag. ([#10](https://github.com/cyanheads/faostat-mcp-server/issues/10))
- **`canvas_id` was ignored.** Both dataframe tools always resolved the session's shared canvas; an explicit `canvas_id` now selects that canvas, and `faostat_dataframe_describe` no longer leaks a different canvas's table metadata. ([#11](https://github.com/cyanheads/faostat-mcp-server/issues/11))
- **`faostat_dataframe_describe` provenance divergence.** Undefined-valued `query_params` keys are stripped at the `stageObservations` write site so `structuredContent` and `content[]` agree (no `key=undefined`); a name-filtered miss now throws typed `missing_table` (`NotFound`) instead of the unqualified "No active staged tables" text. ([#13](https://github.com/cyanheads/faostat-mcp-server/issues/13))
