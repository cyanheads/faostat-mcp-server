# 0.1.4 — 2026-06-19

## Fixed

- **Malformed-column canvas SQL — `faostat_dataframe_query`** — a `SELECT` against a staged table that parses but references a mistyped column (or unknown function / invalid expression) now fails with `data.reason: 'invalid_sql'` and carries the DuckDB binder detail in `data.binderMessage`, naming the offending column. Previously these prepare failures fell through to the gate's `non_select_statement`, where the reason and message pointed away from the real problem. Inherited from `@cyanheads/mcp-ts-core` 0.10.8. ([mcp-ts-core#236](https://github.com/cyanheads/mcp-ts-core/issues/236))
- **`faostat_dataframe_describe` filtered table lookups** — the framework's `DuckdbProvider.describe({ tableName })` no longer raises an ambiguous-column Binder Error; the pushed `WHERE` filters are now qualified with the joined table's alias. Inherited from `@cyanheads/mcp-ts-core` 0.10.7. ([mcp-ts-core#235](https://github.com/cyanheads/mcp-ts-core/issues/235))

## Changed

- **`canvas-staging.ts` SQL gate-reason remap** — the remap that normalizes gate rejections to the tool's declared `invalid_sql` is retained for the genuinely non-SELECT / denied / malformed-identifier reasons (`non_select_statement`, `multi_statement`, denied function/operator). Since mcp-ts-core 0.10.8 a SELECT-shaped prepare failure is classified natively as `invalid_sql` with `data.binderMessage`, so it passes through untouched — re-wrapping it would strip the binder detail. Comments corrected; the regression test now also asserts the native `binderMessage` survives. No change to the tool's advertised `errors[]`.
- **`devcheck.config.json`** — `outdated.allowlist` reconciled to empty; the previously held `@cyanheads/mcp-ts-core`, `@duckdb/node-api`, and `@types/node` are all current after this refresh.
- **Framework scaffold scripts re-synced** — `scripts/devcheck.ts`, `scripts/build-changelog.ts`, and `scripts/check-framework-antipatterns.ts` adopt the 0.10.7 fresh-scaffold guards (git-dependent checks and changelog sync skip cleanly outside a built repo); `scripts/check-skill-versions.ts` skips a worktree-deleted `SKILL.md`. ([mcp-ts-core#242](https://github.com/cyanheads/mcp-ts-core/issues/242), [#243](https://github.com/cyanheads/mcp-ts-core/issues/243), [#237](https://github.com/cyanheads/mcp-ts-core/issues/237))
