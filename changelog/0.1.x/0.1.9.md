# 0.1.9 — 2026-07-13

## Added

- **`faostat_resolve_codes` pagination.** New optional `offset` input (default 0) and `nextOffset` enrichment field (present only when `truncated`) let a caller page through a full area/item/element listing without gaps — previously `truncated: true` gave no way to reach the omitted tail. Truncated and empty-result notices now report the current page range and the offset for the next call. ([#7](https://github.com/cyanheads/faostat-mcp-server/issues/7))

## Fixed

- **`faostat_resolve_codes` cross-domain code leakage.** Item and element resolution is now scoped to the codes actually present in the requested domain's observation cube — previously the shared dimension vocabulary let resolution return (and validate) a code belonging to a different domain, e.g. resolving `RL` against "nitrogen" returned item `3102`, which only exists in `RFN` and returns zero rows from `faostat_query_observations` on `RL`. `FaostatMirror.resolve()` now takes `domain` and threads a domain-membership set (`SELECT DISTINCT item_code|element_code FROM obs_<domain>`, index-backed — no re-sync needed) into `DimensionsStore.resolve()`'s exact-code/FTS/LIKE/list-all branches as a SQL predicate; areas remain unscoped since country vocabularies legitimately overlap domains. `faostat_commodity_profile`'s internal item resolution is now scoped to QCL, its production backbone. ([#8](https://github.com/cyanheads/faostat-mcp-server/issues/8))
