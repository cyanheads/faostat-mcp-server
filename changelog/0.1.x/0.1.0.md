# 0.1.0 — 2026-06-13

## Added

- **`faostat_list_domains`** — reads the live FAOSTAT bulk manifest for the full domain catalog and annotates each entry with local mirror status (`indexed`, `index_ready`, row count, last sync). The entry point — every query keys on a domain code.
- **`faostat_resolve_codes`** — resolves human terms to the opaque area/item/element codes the cube needs, via FTS5 over the bundled dimension code lists. Flags each area `country` vs `aggregate` and surfaces the CPC crosswalk for items.
- **`faostat_query_observations`** — queries a domain's cube by area/item/element codes and year range. Excludes aggregate regions (codes ≥ 5000) by default so naive sums don't double-count; spills large result sets to a DataCanvas table.
- **`faostat_commodity_profile`** — workflow tool assembling top producers, the production trend, and trade flows for one commodity from the QCL + TCL domains in one call; returns a partial result when trade is not indexed.
- **`faostat_dataframe_query` / `faostat_dataframe_describe`** — read-only SQL over the canvas tables staged by the analytical tools, plus staged-table introspection.
- **`faostat-mirror` service** — one `MirrorService` (embedded SQLite + FTS5) per selected domain plus a shared dimension store, fed by a streaming bulk-ZIP ingester (manifest discovery, fflate stream-parse of the normalized data CSV and bundled code lists, M49/CPC apostrophe stripping). `mirror:init` / `mirror:refresh` / `mirror:verify` CLI scripts; `FAOSTAT_DOMAINS`-driven selection; HTTP-only scheduled refresh.

## Notes

- Data path is the keyless bulk-download service (the REST query API is auth-gated); the mirror is the data path, not an optimization.
- Mirror (`better-sqlite3` on Node, `bun:sqlite` on Bun) and DataCanvas (`@duckdb/node-api`) are native — Node/Bun only, no Cloudflare Workers build.
- Run `bun run mirror:init` to load the corpus before querying; the read tools surface `index_not_ready` until the first sync completes.
